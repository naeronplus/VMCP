# E020 — DEAD_LETTER

Job failed max attempts (default 3). Appears in the dead-letter queue; operators may retry or archive after remediation.

## Notification path (H-14 / M-03)

| Stage | Who is emailed | Severity |
|-------|----------------|----------|
| Job enters DLQ | BullMQ `pgos-dead-letter` consumer loads job + project `admin_contacts`, sends **enriched** alert. `ADMIN_EMAIL` is CC (or sole fallback if no contacts). | high |
| Unresolved 24h | Hourly `dead-letter-escalate` → project contacts + `ADMIN_EMAIL` CC | high |
| Unresolved 72h | Same path | critical |

Webhook / audit log still fire on entry from `job-service` (email deferred to the consumer so operators get one enriched message).

## Operator actions

1. `GET /api/v1/dead-letter` — inspect reason, attempts, age  
2. `POST /api/v1/dead-letter/:jobId/retry` (admin) — re-queue  
3. Fix root cause (runner, reimport, fencing, secrets) before retry  
4. Ensure project `admin_contacts` is set so on-call receives mail
