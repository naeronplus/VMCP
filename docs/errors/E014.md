# E014 — PATH_TRAVERSAL

Path input escaped the allowed project base. Request rejected; audit the caller token.

## Typical sources

| Surface | When E014 is raised |
|---------|---------------------|
| Orchestrator APIs | Path/query outside project root (`assertWithinBase`) |
| Structural merge (`POST /merge`) | Relative path escape before local write |
| commit-agent verbs | `ValidateTarget` rejects paths outside `--project-root` |

## commit-agent `merge-apply` (H-02)

Remote structural merge uses the same path rules as other ForcedCommand verbs:

- `project_root` and resolved `rel_path` must stay under agent `--project-root`.
- `rel_path` must be relative (no `..`, no absolute paths).
- Failures surface as **non-zero exit** with an **`E014:`** prefix on **stderr** only (stdout is reserved for success JSON).

### Operator actions

1. Confirm project metadata `projectRoot` / `targetProjectRoot` matches the path on the **target** host (not a path that only exists on the Tier A runner).
2. Confirm commit-agent is installed with `--project-root` (or default `/var/godot/projects`) covering that tree.
3. Inspect `merge_outbox.detail` for the failed row and re-dispatch after fixing paths (`PGOS_MERGE_OUTBOX_REDISPATCH=1` when enabled).
4. Do **not** widen agent roots to `/` — fix the logical path instead.

### Related (not E014)

| Code | Meaning on merge-apply path |
|------|-----------------------------|
| **E019** | Patch introduces executable `script` attachment — always rejected on target agent (admin-only on orchestrator). |
| Workflow / SSH failure | Missing `TARGET_HOST` after resolve, JIT provision failure, or empty agent stdout — outbox stays `dispatched`/`failed` with detail; check runner logs (no PEM). |

Success response (not an error): single stdout line  
`{"ok":true,"mergedHash":"<sha256>","path":"<rel>"}` then runner `POST /merge-outbox/:id/complete`.
