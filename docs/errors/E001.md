# E001 — DISPATCH_TIMEOUT

**Severity:** High

No GitHub Actions runner picked up the `workflow_dispatch` within the configured timeout (default 60s).

## Typical causes

- Self-hosted `godot-worker` offline (Tier A)
- GitHub Actions outage or queue saturation (Tier B)
- Workflow file rename / permissions on GitHub App

## Operator actions

1. Check runner availability and labels.
2. Confirm PGOS retries the alternate tier automatically.
3. After 3 consecutive failures, investigate GitHub App installation token scopes (`actions:write` for dispatch, `actions:read` for polling).
