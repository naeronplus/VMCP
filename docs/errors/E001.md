# E001 — DISPATCH_TIMEOUT / dispatch failure

**Severity:** High

GitHub Actions dispatch did not complete successfully. Covers **runner pickup timeout** and **workflow_dispatch API failure**.

| Path | Job status | Typical cause |
|------|------------|---------------|
| No runner within `DISPATCH_TIMEOUT_MS` | `DISPATCH_TIMEOUT` | Tier offline, queue saturation |
| `createWorkflowDispatch` throws | `DISPATCH_FAILED` | Bad App creds, wrong workflow file/name |
| Local mock (M-18) | `DISPATCH_FAILED` | `GITHUB_MOCK=true` **and** `GITHUB_MOCK_DISPATCH_FAIL=true` |

Both statuses are **retriable** until max attempts → dead-letter (E020).

## Typical causes

- Self-hosted `godot-worker` offline (Tier A)
- GitHub Actions outage or queue saturation (Tier B)
- Workflow file rename / permissions on GitHub App
- Local: intentional mock failure for E2E

## Local E2E (M-18)

```bash
# .env (development)
GITHUB_MOCK=true
GITHUB_MOCK_DISPATCH_FAIL=true
# Optional: mock completed-run conclusion for getRunStatus consumers
GITHUB_MOCK_RUN_CONCLUSION=failure
```

Enqueue a job; after dispatch attempt the job should enter **`DISPATCH_FAILED`** with detail containing `GITHUB_MOCK_DISPATCH_FAIL`.

## Operator actions

1. Check runner availability and labels.
2. Confirm PGOS retries the alternate tier / requeue automatically for retriable dispatch statuses.
3. After 3 consecutive failures, investigate GitHub App installation token scopes (`actions:write` for dispatch, `actions:read` for polling).
4. For local-only mock failures, clear `GITHUB_MOCK_DISPATCH_FAIL`.
