# E004 — COMMIT_FAILED

Atomic commit failed: fencing token rejected, agent error, transfer checksum mismatch, or **cross-machine pre-commit S3 snapshot failure** (C-03).

## Common cross-machine `errorDetail` values (C-03)

| Detail | Cause |
|--------|--------|
| `pre-commit snapshot-export failed` | Target `snapshot-export` verb failed (path missing, agent error, SSH) |
| `pre-commit snapshot-export failed: empty archive` | Verb succeeded but stdout archive was empty |
| `pre-commit snapshot S3 upload failed` | `pgos_upload_file` / presigned PUT failed |
| `pre-commit snapshot presign missing (PRESIGN_SNAPSHOT_PUT)` | Worker secrets envelope lacked snapshot PUT URL |

Cross-machine **never** snapshots runner-local `TARGET_ROOT`. Primary rollback source is the S3 archive produced by target `snapshot-export` before `stage-receive`.

## Operator actions

1. Inspect lock history (`GET /locks`).
2. If stale, admin reclaim via `POST /locks/reclaim` and redispatch.
3. For cross-machine, verify commit-agent service and JIT SSH provisioning.
4. For C-03 snapshot failures: confirm target project path exists under `--project-root`, `snapshot-export` is installed, and S3 presigns are present in `resolve-secret`.
