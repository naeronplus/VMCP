# E021 — INVALID_STATUS_TRANSITION

The worker (or internal caller) requested a job status that is not allowed from the current FSM state (HTTP **409**).

Examples:

- `QUEUED` → `COMPLETED` (skip-ahead)
- `STAGING` → `COMPLETED`
- `REIMPORT_FAILED` → `COMMITTING` / `COMPLETED` (commits blocked until re-dispatch)

## Not E021

| Code | Meaning |
|------|---------|
| **E019** | Merge/override introduces executable scripts and the actor is not **admin** (`SCRIPT_OVERRIDE_REQUIRES_ADMIN`). Never used for FSM transitions. |
| **E013** | Fencing token rejected on status/heartbeat |
| **E015** | Auth / wrong role or token scope |

## Operator actions

1. Inspect current job status (`GET /api/v1/jobs/:id`) and the worker callback sequence.
2. Confirm the worker only advances along allowed edges (see `JOB_STATUS_TRANSITIONS` in `@vibrato/shared`).
3. If the job is stuck after reclaim or failure, use admin lock reclaim or dead-letter retry — do **not** force an illegal status with an operator JWT (`PATCH .../status` is callback-only).
4. Deep link: `/api/v1/docs/errors/E021` (dashboard Error catalog).
