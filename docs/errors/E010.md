# E010 — PARITY_FAILURE

Tier A and Tier B canary outputs differ (checksum mismatch), Godot reimport failed on a canary tier (`reimport_failed_a` / `reimport_failed_b` / `reimport_failed_both`), or Tier B was unavailable.

Diff artifact is uploaded when available; consider disabling the offending tier.

## Not E010

| Situation | Behavior |
|-----------|----------|
| Tier A self-hosted runner unavailable | Canary **skips** with `skipped: true`, `reason: tier_a_unavailable`. Recorded on `parity_checks` but **no E010 alert**. |
| Parity checks stale (>48h) | Separate stale-scan alert (still code E010 historically) — distinct from a failed A/B compare. |

## Operator actions

1. Inspect latest `parity_checks.reason` and dashboard Tiers page.
2. If `reimport_failed_*`: check Godot install on the failing tier (`setup-godot.sh`, export templates).
3. If `checksum_mismatch`: download canary artifacts and compare project trees.
4. If only skips (`tier_a_unavailable`): restore Tier A `godot-worker` self-hosted runner; skips alone are not failures.
