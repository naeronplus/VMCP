# AGENTS.md — Godot PGOS / Vibrato

Onboarding for agents and operators working with the **Procedural Generation Orchestration Service** (MCP name: **Vibrato**).

## Architecture reminders

- PGOS **never runs Godot**. Godot runs only on GitHub Actions runners (Tier A self-hosted `godot-worker` or Tier B hosted).
- **Push, not pull:** jobs are started with `workflow_dispatch`.
- **Artifacts:** all worker inputs/outputs go through **S3**. Railway volumes are orchestrator-local cache only.
- **Locks:** Redis is source of truth for lock validity; Postgres `lock_fencing_seq` is the authoritative fencing ledger for commits.
- **Redis failover (M-17):** `LockService.rotateInstanceIdOnFailover` rotates the composite-token `instanceId` **and** INSERTs `lock_fencing_seq` rows with `reason='FAILOVER'` (per active `pg_locks` key + sentinel `pgos:system:redis-failover`). `validateFencingToken` rejects when the latest reason is `FAILOVER` (or instanceId mismatches). Holders must re-acquire.

## Headless Godot setup

```bash
godot --headless --editor --quit --path /path/to/project
```

This forces reimport, populates `.godot/imported/`, and surfaces `uid://` resolution errors.

### Version matching

- Dispatch always includes `godotVersion` (e.g. `4.3.1`).
- Worker runs `workers/scripts/verify-godot.sh` for **exact** major.minor.patch equality (not substring match — `4.3.1` ≠ `4.3.10`) and export template presence (E006).
- Export templates must match the editor version under `export_templates/{version}.stable/`.

### Cache keys

GitHub Actions cache key: `godot-{version}-{platform}`. On checksum mismatch, re-download from the official Godot mirror.

## UID system

| Prefix | Meaning |
|--------|---------|
| `GEN-` | Generated by PGOS |
| `OVRD-` | User override (canonical in conflicts) |
| `TMP-` | Reservation placeholder before commit |
| `USER-` | External / manual |

### Avoiding collisions

1. Before writing a new asset, call `POST /projects/:id/uid-reservations` with the logical path.
2. Generate content using the reserved temporary UID.
3. Commit final UID via `POST /projects/:id/uid-reservations/commit`.

Concurrent reservations for the same logical path are serialized with row-level locks.

### External edits

Nightly reconciliation (`pgos-uid-reconcile` worker):

1. Finds duplicate UIDs in `uid_mappings` (canonical = OVRD- or newest).
2. When `project_root` is readable on the orchestrator, scans `.tscn`/`.tres`/`.import`/… for `uid://` tokens and rewrites full tokens only.
3. Runs headless Godot reimport when files change; failures raise **E008** for manual review.
4. When files live only on a remote host **and** project `metadata.targetHost` (and optionally `targetProvisionUrl` / `uidReconcileUrl`) is set, nightly reconcile **auto-dispatches** `uid_reconcile.yml` with a replacements map on S3 (`mode: remote_dispatched`). Without host metadata, the orchestrator audits `mode: remote_script` for manual `workers/scripts/uid-reconcile.sh`.

## Common pitfalls

1. **Editing `.tscn` by hand** without understanding `ext_resource` / UID links — prefer Godot editor or structural merge tools.
2. **Copy-pasting scenes** creates duplicate UIDs — use the reservation API or let nightly reconcile fix them.
3. **Opening the project in the editor during commit** creates `project.godot.lock`; workers wait up to 5 minutes then pause (E012).
4. **Putting secrets in workflow inputs** — dispatch passes only `secretJwe`. The callback credential is embedded inside the JWE; workers call `POST /resolve-secret` with `{ jwe }` only (no separate `callbackToken` workflow input).

## Structural merge for `.tscn`

Overrides applied via `POST /merge` perform structural merge when enabled (`PGOS_STRUCTURAL_MERGE` default on; set `0` to disable file writes):

1. Match nodes by stable path / name within parent (`Root/Player`).
2. Merge properties; `ext_resource` rows merge by `uid://` when present.
3. **Script attachments** (`script = ExtResource(...)`) require **admin** role (threat model E019).
4. Never invent new executable scripts from operator-tier patches.
5. **Apply modes:**
   - `local_fs` (auto when `project_root` is readable on the orchestrator): atomic write of merged `.tscn` + override row + content hash.
   - `outbox`: override row + `merge_outbox` pending row when the tree is **not** on the orchestrator. **Automatic consumer** (`pgos-merge-outbox` every 5m): applies locally when root becomes readable; otherwise **remote apply** (below). Not a manual outbox.
   - Force with `PGOS_MERGE_MODE=local_fs|outbox|auto|registry_only`.

### Remote apply path (H-02 — not co-location-only)

When `project_root` is unreadable on the orchestrator and project metadata includes `targetHost` (and usually `targetProvisionUrl`):

1. Consumer uploads patch JSON to S3 (`projects/{id}/merge-outbox/{outboxId}/patch.json`) and presigns GET.
2. Orchestrator builds a **dispatch JWE** (`secretJwe`) sealing `callbackToken` / service token, optional JIT `sshPrivateKey`, `targetHost`, `targetProjectRoot`, `outboxId`, `relPath`, `patchGetUrl`, `pgosBaseUrl`. **No raw private keys in workflow inputs.**
3. Dispatches **`merge_apply.yml`** (`workflow_dispatch` only) on Tier A (`self-hosted`, `godot-worker`).
4. Runner: `resolve-secrets.sh` → env (`TARGET_HOST`, `CALLBACK_TOKEN`, key file) → `workers/scripts/merge-apply.sh`.
5. If `PROJECT_ROOT` is not a local directory: require `TARGET_HOST`; pipe patch via **`pgos_ssh_agent_stdin "merge-apply <project_root> <rel_path>"`** (commit-agent verb on the **target** host). Never raw `ssh`/`scp`.
6. Agent stdout JSON `{"ok":true,"mergedHash":"…","path":"…"}` → runner `POST /api/v1/merge-outbox/:id/complete` → outbox `status=applied`.

Orchestrator **never** writes remote `.tscn` files itself (PGOS never runs Godot; merge-apply is target-side only). Failures: path escape **E014**; script patches on target **E019**.

### Example

```
Base:   Root/Player (script: player.gd)
Patch JSON: { "nodes": [{ "path": "Root/Player", "properties": { "position": "Vector2(10, 0)" } }] }  # operator OK
Patch with script property → admin only (E019)
```

## Export templates

1. Install templates matching exact editor version.
2. Worker setup validates version before generation.
3. On mismatch, job fails with E006 — update templates, do not force export.

## Troubleshooting reimport (E002)

| Symptom | Check |
|---------|--------|
| Exit non-zero | `reimport_failure.log` in S3 diagnostics |
| Missing modules / custom builds | Ensure CI image has required tools (`scons` only if compiling modules) |
| UID errors in log | Run UID reconcile; fix dangling `ext_resource` |
| Post-commit failure | Automatic rollback from S3 snapshot; inspect diff |

## Error deep links

All codes E001–E021 document operator actions under `/docs/errors/{code}` and the dashboard **Error catalog** page (E019 = script override admin only; E021 = invalid job status transition).

## Callback protocol

Workers authenticate with short-lived `callbackToken` (5 minutes, job-scoped):

- `PATCH /api/v1/jobs/:id/status`
- `PATCH /api/v1/jobs/:id/heartbeat` every 15s
- `POST /api/v1/resolve-secret` once for the JWE envelope

Never embed long-lived credentials in runner logs.
