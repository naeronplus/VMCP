# AGENTS.md — Godot PGOS / Vibrato

Onboarding for agents and operators working with the **Procedural Generation Orchestration Service** (MCP name: **Vibrato**).

## Architecture reminders

- PGOS **never runs Godot**. Godot runs only on GitHub Actions runners (Tier A self-hosted `godot-worker` or Tier B hosted).
- **Push, not pull:** jobs are started with `workflow_dispatch`.
- **Artifacts:** all worker inputs/outputs go through **S3**. Railway volumes are orchestrator-local cache only.
- **Locks:** Redis is source of truth for lock validity; Postgres `lock_fencing_seq` is the authoritative fencing ledger for commits.

## Headless Godot setup

```bash
godot --headless --editor --quit --path /path/to/project
```

This forces reimport, populates `.godot/imported/`, and surfaces `uid://` resolution errors.

### Version matching

- Dispatch always includes `godotVersion` (e.g. `4.3.1`).
- Worker must run `godot --version` and **fail fast** on mismatch (E006).
- Export templates must match the editor version.

### Cache keys

GitHub Actions cache key: `godot-{version}-{platform}`. On checksum mismatch, re-download from the official Godot mirror.

## UID system

| Prefix | Meaning |
|--------|---------|
| `GEN-` | Generated by PGOS |
| `OVRD-` | User override (canonical in conflicts) |
| `TMP-` | Reservation placeholder before commit |
| `USER-` | External / manual |

### Avoiding collisions

1. Before writing a new asset, call `POST /projects/:id/uid-reservations` with the logical path.
2. Generate content using the reserved temporary UID.
3. Commit final UID via `POST /projects/:id/uid-reservations/commit`.

Concurrent reservations for the same logical path are serialized with row-level locks.

### External edits

Nightly reconciliation scans project files for `uid://` strings, auto-resolves duplicates (canonical = OVRD- or newest), and rewrites references via headless Godot when possible. Failures raise E008 for manual review.

## Common pitfalls

1. **Editing `.tscn` by hand** without understanding `ext_resource` / UID links — prefer Godot editor or structural merge tools.
2. **Copy-pasting scenes** creates duplicate UIDs — use the reservation API or let nightly reconcile fix them.
3. **Opening the project in the editor during commit** creates `project.godot.lock`; workers wait up to 5 minutes then pause (E012).
4. **Putting secrets in workflow inputs** — dispatch passes only `secretJwe`. The callback credential is embedded inside the JWE; workers call `POST /resolve-secret` with `{ jwe }` only (no separate `callbackToken` workflow input).

## Structural merge for `.tscn`

Overrides applied via `POST /merge` perform structural merge:

1. Match nodes by stable path / name within parent.
2. Merge properties; sub-resources merge by UID when present.
3. **Script attachments** (`script = ExtResource(...)`) require **admin** role (threat model E019).
4. Never invent new executable scripts from operator-tier patches.

### Example

```
Base:   Root/Player (script: player.gd)
Patch:  Root/Player { position: Vector2(10, 0) }   # operator OK
Patch:  Root/Player { script: enemy.gd }           # admin only
```

## Export templates

1. Install templates matching exact editor version.
2. Worker setup validates version before generation.
3. On mismatch, job fails with E006 — update templates, do not force export.

## Troubleshooting reimport (E002)

| Symptom | Check |
|---------|--------|
| Exit non-zero | `reimport_failure.log` in S3 diagnostics |
| Missing modules / custom builds | Ensure CI image has required tools (`scons` only if compiling modules) |
| UID errors in log | Run UID reconcile; fix dangling `ext_resource` |
| Post-commit failure | Automatic rollback from S3 snapshot; inspect diff |

## Error deep links

All codes E001–E020 document operator actions under `/docs/errors/{code}` and the dashboard **Error catalog** page.

## Callback protocol

Workers authenticate with short-lived `callbackToken` (5 minutes, job-scoped):

- `PATCH /api/v1/jobs/:id/status`
- `PATCH /api/v1/jobs/:id/heartbeat` every 15s
- `POST /api/v1/resolve-secret` once for the JWE envelope

Never embed long-lived credentials in runner logs.
